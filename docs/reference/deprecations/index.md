# Deprecations and replacements

Each entry must include:

- deprecated route, field, enum, command, configuration key, or behavior;
- replacement and migration steps;
- first deprecated and removal release or date;
- compatibility behavior and warnings;
- redirect or alias lifetime;
- accountable owner.

A documentation move alone is not a product deprecation. Legacy documentation URLs are tracked in the Phase 9 redirect map.
