# Preview and validate documentation

1. Run the strict MkDocs build for the reviewed configuration.
2. Check internal links, anchors, assets, generated facts, IA uniqueness, and redirects.
3. Inspect landing, task, troubleshooting, dense reference, and contributor pages at mobile and desktop widths.
4. Test keyboard navigation, visible focus, landmarks, headings, search, drawer behavior, 200% zoom, and reduced motion.
5. Run representative natural-language search and reader tasks.
6. Compare custom code and CI cost with the complexity budget.
7. Attach the built artifact or preview and concise evidence to the pull request.

Do not use screenshot hashes, exact-prose assertions, or broad decorative visual regression as launch gates.
