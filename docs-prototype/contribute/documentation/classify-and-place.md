# Classify and place a page

1. State the exact task, concept, symptom, or lookup.
2. Check whether a canonical page already owns it.
3. Choose task, workflow, concept, troubleshooting, runbook, reference, architecture, deprecation, or landing type.
4. Place it under the domain that owns the reader outcome.
5. Assign one canonical URL, owner, source of truth, applicability, and lifecycle.
6. Record aliases or redirects for replaced pages.
7. Keep internal plans, QA evidence, and rollout records outside the public source.

Cross-links expose related contexts; they do not create a second navigation placement.
