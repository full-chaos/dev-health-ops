# Feature flags and availability

For each supported feature, generate or verify:

- stable key and current label;
- owner and affected route or service;
- default and environment or entitlement behavior;
- prerequisite source, role, or deployment capability;
- fallback when disabled;
- review or removal trigger.

A route marked preview or guarded by a required feature must not be documented as generally available. Context Fabric remains reserved for a later customer-task IA amendment.
