# Planning

Planning views turn selected backlog, throughput history, and product trends into explicit scenarios for a team conversation. They describe uncertainty in a particular system and period; they are not commitments, staffing formulas, or individual performance targets.

## Choose the right planning input

Use Completion Forecast when the question concerns a defined backlog and possible completion windows. Use trend comparison when the question concerns how a measure or work mix changes across equivalent periods. Return to the relevant product view and supporting work when the planning discussion needs an explanation for the pattern.

A forecast becomes less representative when scope, work size, staffing, workflow, or source coverage changes materially. A trend comparison becomes misleading when the populations or units differ. State those changes alongside the result rather than hiding them inside one number.
