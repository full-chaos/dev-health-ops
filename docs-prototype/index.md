# Find the task in front of you

Dev Health documentation should tell you what to do, what to expect, and where to recover when the result is incomplete. It should not make you learn the repository or the documentation project.
{: .fc-lede }

This is a layout and style prototype. It does not make the current WIP content canonical.
{: .fc-prototype-note }

## Choose a domain

- [**Get started**](get-started/index.md)<br>Check the minimum prerequisites and choose a real task. This provisional branch uses entirely new content.
- [**Use Dev Health**](use/index.md)<br>Use workflows, interpret results, create reports, and recover from user-visible problems.
- [**Administer Dev Health**](admin/index.md)<br>Configure the workspace, access, sources, synchronization, and administrative security.
- [**Install and operate**](operate/index.md)<br>Deploy, upgrade, monitor, maintain, troubleshoot, and recover the platform.
- [**Integrate and extend**](integrate/index.md)<br>Send data, use supported APIs and webhooks, and build supported integrations.
- [**Reference**](reference/index.md)<br>Look up exact API, CLI, configuration, schema, metric, taxonomy, and compatibility facts.
- [**Contribute**](contribute/index.md)<br>Set up development, understand boundaries, test changes, release, and improve documentation.
