# Version and compatibility matrix

The release-specific matrix should identify:

- application and web compatibility;
- Postgres, ClickHouse, Redis/Valkey, and migration compatibility;
- deployment artifact and container architecture support;
- public schema and API versions;
- provider and self-managed-host support;
- required feature or entitlement;
- upgrade and rollback constraints;
- deprecated and removed versions.

Do not infer support from a file remaining in the repository. Link every claim to a current release, test, or source contract.
